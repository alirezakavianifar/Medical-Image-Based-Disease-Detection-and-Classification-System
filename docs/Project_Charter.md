# Project Charter: Medical Image-Based Disease Detection and Classification System

## 1. Project Title
**Medical Image-Based Disease Detection and Classification System (Documentation & Process Lifecycle)**

---

## 2. Project Objectives
The primary goal of this project is to produce a complete suite of professional, standards-compliant software engineering documentation and establish a robust, collaborative version-control lifecycle.
Specifically, the project aims to:
- Establish a Software Requirements Specification (SRS) in alignment with IEEE 830 standards to capture user needs for medical image upload, disease analysis, and reporting.
- Design a Software Design Document (SDD) containing comprehensive UML architectural diagrams (Use Case, Activity, Class, Sequence) and database schema models.
- Build thorough User/Installation Guides and Test Documentation to verify system features and facilitate system deployment.
- Implement structured version control, quality review workflows, and pull request conventions.

---

## 3. Project Scope

### 3.1 In-Scope
The project scope encompasses all documentation, design, and management aspects of the product lifecycle:
- **Requirements Engineering:** Gathering, structuring, and validating functional requirements (e.g., image upload, AI analysis, doctor dashboards) and non-functional requirements (e.g., accuracy, system performance, safety).
- **System Architecture Design:** Mapping the application tiers (Presentation, Business Logic, AI Detection, Database) and specifying API endpoints.
- **Verification planning:** Writing detailed test cases, executing tests, and producing test reports.
- **Project and Git Configuration:** Establishing the directory layout, git branch structure, PR review criteria, and tagging conventions.

### 3.2 Out-of-Scope
- **AI/ML Model Training:** Gathering raw medical datasets (e.g., Chest X-Rays, Brain MRIs) and coding/training neural networks.
- **Backend/Frontend Implementation:** Writing production-grade code for servers or web clients.
- **Medical/Clinical Certification:** Obtaining official clearance (e.g., FDA, CE mark) for clinical usage, though regulatory compliance constraints are documented.

---

## 4. Stakeholders
- **Client/Hospital Representative:** Defines the clinical requirements and operational constraints of the hospital environment.
- **Project Supervisor:** Reviews and approves deliverables, ensuring academic/professional standards are met.
- **Development Team:** Uses the SRS and SDD to understand implementation logic (represented theoretically in this lifecycle).
- **Quality Assurance (QA) Team:** Executes manual and automated tests according to the Test Documentation.
- **End Users (Doctors, Radiologists):** Primary users who interact with the system interface to analyze images and view reports.

---

## 5. Project Deliverables
1. **Software Requirements Specification (SRS):** Complete functional and non-functional requirements.
2. **Software Design Document (SDD):** High-level and detailed design, UML diagrams, database schemas, and API design.
3. **User, Installation, and Test Documentation:** User manual, installation guides, test cases, and test run reports.
4. **Git Repository History:** Version-controlled development history showing branch management and peer reviews.
5. **Final Project Report:** Detailed final summary of activities, challenges, lessons learned, and project status.

---
